# 需求分析

## 任务目标

本项目面向通用医学教材知识整合场景，而不是只对七本测试教材写固定规则。系统需要接收多格式教材，解析为统一结构，抽取章节级正文，再进一步完成知识点抽取、跨教材关系归并和基于原文的问答。

## 子问题拆解

1. 教材加载与解析：支持 PDF、Markdown、TXT、DOCX，输出统一 JSON。
2. 正文清洗：去除封面、目录、页眉页脚、二维码数字资源提示、索引等非教学正文。
3. 章节识别：从页文本中识别章标题、起止页、正文内容和字数。
4. 知识点抽取：将章节内容交给大模型抽取概念、定义、类别和来源片段。
5. 知识融合：对跨章节、跨教材重复概念进行归一化合并。
6. 关系建模：保留先修、并列、包含、应用等教学关系。
7. RAG 问答：在原文片段中检索证据，生成带来源引用的答案。
8. 可视化交互：提供图谱拖拽、缩放、筛选和教材切换。

## 知识点粒度定义

知识点以“可独立教学、可被定义或解释、可作为问答检索单元”为最小粒度。系统优先抽取：

- 概念：如疾病、结构、功能、机制、实验方法。
- 过程：如病理变化、感染机制、代谢调控。
- 分类：如疾病分类、组织分类、功能分类。
- 临床或实验联系：如症状、检查、治疗原则。

过细的孤立短语和过宽的整章主题都会降低图谱可用性，因此抽取提示要求模型给出定义和来源片段，用于约束粒度。

## 重复判定标准

重复上传判定使用 SHA-256 文件指纹。同一文件重复上传时直接复用已有解析结果，不创建重复教材。

知识点重复判定使用规范化名称：去除空白和常见标点，统一大小写，再按名称键合并。合并时保留首次出现的定义、章节、页码和来源片段。该策略避免简单同名概念重复展示，同时保持实现可解释。

## 教学连续性保障

章节解析阶段以页码顺序建立章范围，正文存储仍保留章节完整内容。RAG 分块采用重叠窗口，避免关键定义跨块时被截断。知识图谱关系保留 `prerequisite`、`contains` 等类型，用于表达学习先后与层次关系。

## 压缩率计算

压缩率用于衡量从教材原文到结构化知识表达的压缩效果：

```text
压缩率 = 结构化知识字符数 / 教材正文总字符数
```

结构化知识字符数包含知识点名称、定义、关系描述和来源片段摘要，不包含完整教材正文。完整正文仍保存在解析结果中，用于导出和 RAG 溯源。

## RAG 分块策略

默认分块大小为 600 字，重叠 120 字。原因如下：

- 医学教材句子和定义较长，600 字可以容纳一个局部知识单元。
- 120 字重叠能减少段落边界导致的信息丢失。
- 分块粒度小于整章，检索更精准；又大于单句，模型回答时有足够上下文。
- `top_k` 默认 5，兼顾召回覆盖与提示词长度。

## 边界与风险

- 扫描版 PDF 若没有可提取文本，需要 OCR 扩展。
- 不同出版社的章节排版差异大，规则需要继续积累。
- 大模型抽取结果可能有漏抽或关系类型偏差，需要结合人工复核或二次校验。
- 在线部署环境如果磁盘是临时的，需要挂载持久化存储保存上传教材和索引。
